# Icon System

Vuetify features a powerful, extensible icon system that supports multiple icon libraries including Material Design Icons, Font Awesome, Material Icons, and many more. The system provides automatic icon resolution, alias support, and flexible rendering strategies.

## Icon Configuration

Configure icons during Vuetify initialization (`icons.ts:22`):

```typescript
import { createVuetify } from 'vuetify'
import { aliases, mdi } from 'vuetify/iconsets/mdi'
import { fa } from 'vuetify/iconsets/fa'

const vuetify = createVuetify({
  icons: {
    defaultSet: 'mdi',
    aliases,
    sets: {
      mdi,
      fa,
    },
  },
})
```

### IconOptions Interface

The icon configuration accepts these options (`composables/icons.tsx:84-91`):

```typescript
type IconOptions = {
  defaultSet?: string
  aliases?: Partial<IconAliases>
  sets?: Record<string, IconSet>
}
```

  <Tab title="VLigatureIcon">

### Icon Resolution Logic

The resolution process (`composables/icons.tsx:193-231`):

1. **Alias resolution** - If icon starts with `$`, resolve from aliases
2. **Type detection** - Determine if icon is array (SVG), component, or string
3. **Set detection** - Check if icon contains `set:` prefix
4. **Component selection** - Return appropriate icon component

```typescript
// Alias resolution
if (icon.startsWith('$')) {
  icon = icons.aliases?.[icon.slice(1)]
}

// Array = SVG
if (Array.isArray(icon)) {
  return { component: VSvgIcon, icon }
}

// Component
if (typeof icon !== 'string') {
  return { component: VComponentIcon, icon }
}

// Set detection
const iconSetName = Object.keys(icons.sets).find(
  setName => icon.startsWith(`${setName}:`)
)

const iconSet = icons.sets[iconSetName ?? icons.defaultSet]
return { component: iconSet.component, icon: iconName }
```

## Best Practices

1. **Use aliases** - Prefer semantic aliases (`$complete`) over specific icons
2. **Choose one primary set** - Use one main icon set for consistency
3. **Tree-shake unused sets** - Only import icon sets you use
4. **Lazy load icon fonts** - Load icon fonts asynchronously when possible
5. **Consider SVG** - Use SVG icon sets for better performance and smaller bundles
6. **Cache CDN fonts** - Configure proper caching for icon font CDN resources
7. **Provide fallbacks** - Ensure icons have text alternatives for accessibility

## Performance Optimization

### SVG Icon Sets

For better performance, use SVG-based icon sets:

```typescript
import { mdiAccount, mdiHome } from '@mdi/js'
import { aliases } from 'vuetify/iconsets/mdi-svg'

const customSvgAliases = {
  ...aliases,
  account: mdiAccount,
  home: mdiHome,
}

const vuetify = createVuetify({
  icons: {
    defaultSet: 'mdi',
    aliases: customSvgAliases,
    sets: {
      mdi: {
        component: VSvgIcon,
      },
    },
  },
})
```

Benefits:
- Smaller bundle size (only icons you use)
- No font loading delay
- Better rendering performance
- Easier to customize

### Icon Subsetting

When using icon fonts, subset to only include needed icons:

```bash
# Example: Generate MDI subset
npx @mdi/font-build --icons home,account,menu --output custom-mdi.css
```

    <v-icon icon="mdi-plus" />
