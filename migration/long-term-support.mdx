# Long-term Support

Vuetify provides Long-term Support (LTS) for major releases to ensure stability and reliability for production applications. This page outlines our LTS policy, support timelines, and what you can expect from each major version.

## LTS Policy

Vuetify follows a structured Long-term Support policy:

- **6 months** of support for major releases
- Bug fixes and security patches during the support period
- Community support via Discord and GitHub
- Commercial support options available for legacy versions

## Current Support Status

### Vuetify 4 (Current)

**Status**: Active Development

- **Version**: 4.0.0
- **Release Date**: 2024
- **Support Level**: Full support
- **Minimum Requirements**:
  - Vue.js 3.5.0+
  - Node.js 24.11.1+
  - TypeScript 4.7+ (optional)

Vuetify 4 is the current major version and receives:

- New features and components
- Bug fixes and performance improvements
- Security patches
- Documentation updates
- Community and commercial support

### Vuetify 3

**Status**: Maintenance

- **Support Level**: Limited (6-month LTS window)
- **Focus**: Critical bug fixes and security patches
- **Migration Path**: Upgrade to Vuetify 4

During the 6-month LTS period, Vuetify 3 receives:

- Critical bug fixes
- Security patches
- Limited community support
- No new features

### Vuetify 2

<Warning>
  Vuetify 2 is now End of Life (EOL) and no longer receives support, even for security issues.
</Warning>

**Status**: End of Life (EOL)

- **Support Level**: None
- **Security Updates**: Not available
- **Community Support**: Limited

Vuetify 2 has reached End of Life and is no longer maintained by the core team. Applications using Vuetify 2 should migrate to a supported version.

**Commercial Support Available**: For organizations that require continued Vuetify 2 support, commercial support is available through our partner:

- **Provider**: [HeroDevs](https://www.herodevs.com/support/vuetify-nes?utm_source=partnership&utm_medium=partnership&utm_campaign=partnership&utm_id=vuetify2)
- **Services**: Security patches, bug fixes, and extended support
- **Ideal For**: Legacy applications that cannot immediately upgrade

## Support Timeline

Here's what to expect during the support lifecycle:

<Steps>

### Active Development (Current Version)

- **Duration**: Until next major release
- **Updates**: Regular releases with new features, improvements, and fixes
- **Frequency**: Patch releases as needed, minor releases quarterly
- **Support**: Full community and commercial support

### Long-term Support (Previous Major Version)

- **Duration**: 6 months from next major release
- **Updates**: Critical bug fixes and security patches only
- **Frequency**: As needed for critical issues
- **Support**: Community support via Discord, limited GitHub support

### End of Life (Legacy Versions)

- **Duration**: After LTS period expires
- **Updates**: None from core team
- **Support**: Community-driven only, commercial options available
- **Recommendation**: Migrate to a supported version

</Steps>

## What's Included in Support

### During Active Development

- Security patches
- Documentation updates
- Migration guides and tools
- Active community support
- Commercial support options

### During LTS Period

- Critical bug fixes

- Limited documentation updates

### After EOL

- No official updates or patches
- Community-driven support only
- Commercial support through partners (e.g., HeroDevs)
- Self-service resources remain available

## Security Policy

### Reporting Security Issues

If you discover a security vulnerability:

1. **Do not** open a public GitHub issue
2. Email the security team with details
3. Allow time for the team to address the issue
4. Coordinate disclosure timing

### Security Updates

- **Current Version**: Immediate security patches
- **LTS Version**: Security patches within 30 days
- **EOL Versions**: No security support (use commercial options)

## Migration Recommendations

### When to Upgrade

Consider upgrading when:

- A new major version is released
- Your current version enters the LTS period
- Security vulnerabilities are discovered
- You need new features or improvements
- You want to benefit from performance optimizations

### Planning Your Migration

For successful migration:

1. **Review Breaking Changes**: Check release notes and migration guides
2. **Test Thoroughly**: Use staging environments to validate changes
3. **Allocate Time**: Plan for testing and potential refactoring
4. **Monitor Dependencies**: Ensure all dependencies are compatible
5. **Seek Support**: Use community resources and documentation

See the [Upgrading to v4](/migration/upgrading-to-v4) guide for detailed migration instructions.

## Frequently Asked Questions

### How long will Vuetify 4 be supported?

Vuetify 4 will receive full support until the next major version (v5) is released, followed by a 6-month LTS period.

### Can I still use Vuetify 2?

Yes, but it's not recommended for new projects. Vuetify 2 is EOL and receives no updates. Consider commercial support if you cannot upgrade immediately.

### What happens after the LTS period ends?

After the LTS period, the version becomes EOL and receives no official support. You should migrate to a supported version or arrange commercial support.

### Are security patches backported?

Security patches are provided for the current version and LTS versions only. EOL versions do not receive security patches unless you have commercial support.

### How do I know when my version enters LTS?

Watch for announcements on:

- The official [Vuetify blog](https://vuetifyjs.com)
- [GitHub releases](https://github.com/vuetifyjs/vuetify/releases)
- [Discord community](https://community.vuetifyjs.com)
- Release notes and documentation
