## Versioning

Vuetify follows [Semantic Versioning](https://semver.org/) (SemVer):

- **Major versions** (e.g., 3.0.0) may contain breaking changes
- **Minor versions** (e.g., 3.1.0) add new features in a backward-compatible manner
- **Patch versions** (e.g., 3.1.1) contain backward-compatible bug fixes

### Long-term Support

Vuetify provides **6 months** of long-term support for major releases. Learn more about our [Long-term Support policy](https://vuetifyjs.com/introduction/long-term-support/).

## Release Workflow

<Steps>
  <Step title="Development">
    New features and bug fixes are developed and tested in feature branches.
  </Step>
  
  <Step title="Beta/RC Releases">
    Major updates go through beta and release candidate phases for community testing.
  </Step>
  
  <Step title="Stable Release">
    Once thoroughly tested, changes are released in a stable version.
  </Step>
  
  <Step title="Documentation">
    Release notes are published with detailed information about changes.
  </Step>
</Steps>
